== Contexte

Le projet api-key-pool etait un build standalone sans separation consommateur/plugin.
Transformation vers l'architecture « Racine Consommateur + Plugin Independant »
decrite dans l'article 0124 du blog cheroliv.com. Renommage du repertoire en
api-key-pool-gradle pour respecter la convention foundry/public/.
